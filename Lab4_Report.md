# Отчёт по Лборторной работе №4
## Задача 
Запустить в двух контейнерах приложение aafire и настроить сеть между этими контейнерами

## Ход работы
### 1. Создал директорию для контйнеров с  помощью ```mkdir```
### 2. Создал докер файл 

Создал Dockerfile со следующим содержимым:

```
FROM ubuntu:24.04

ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update && \
    apt-get install -y --no-install-recommends libaa-bin iputils-ping && \
    rm -rf /var/lib/apt/lists/*

CMD ["aafire"]
```

Пояснение:
- Указал базовый образ для созадания Docker образа
- Установил переменную окружения в значение noninteractive, чтобы пакеты устанвливались самостоятельно и не спрашивали ничего у меня
- Обновил список доступных пакетов
- Установил нужные пакеты в образ, без установления необязательных зависимостей и с флагом -y, чтобы автоматически отвечать да, на все появляющиеся вопросы
- Удалил временные файлы файлового менеджера, для уменьшения размера докер-образа
- Написал команду запуска приложения aafire, которая выполняется при запуске контейнера

### 3. Собрал образ с помощью команды ```docker build -t my-aafire:1.0 .```, где тег я придумал свой
### 4. Проверил, что образ собран с помощью команды ```docker images```
### 5. Запустил контейнеры

- В терминале А запустил контейнер aafire1
- В терминале Б запустил контейнер aafire2

### 6. Настроил связь между контейнерами

- Запсутил третий терминал для создания сети
- В нём созал сеть с помощью команды ```docker network create```
- Подключил запущенные контейнеры к этой сети с помощью ```docker network connect```

- Подключился к первому контейнеру с помощью ```docker exec```
